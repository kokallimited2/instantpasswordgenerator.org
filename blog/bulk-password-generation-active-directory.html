Bulk Password Generation for Windows Active Directory: A Complete Automation Guide

7 May 2026·9 min·Alex Chen, IT Security Specialist

Bulk password provisioning is one of the most operationally sensitive tasks in IT administration. Done correctly it is a 20-minute operation with a clean audit trail.

Step 1: Generate Passwords via CSPRNG

Use Instant Password Generator with length 16+, all character classes, CSV format. Or generate directly in PowerShell:

function New-SecurePassword {
    param([int]$Length = 16)
    $chars = 'ABCDEFGHJKLMNPQRSTUVWXYZabcdefghjkmnpqrstuvwxyz23456789!@#$'
    $rng   = [System.Security.Cryptography.RNGCryptoServiceProvider]::new()
    $bytes = [byte[]]::new($Length); $rng.GetBytes($bytes)
    return -join ($bytes | ForEach-Object { $chars[$_ % $chars.Length] })
}
1..100 | ForEach-Object {
    [PSCustomObject]@{Index=$_; Password=New-SecurePassword -Length 16}
} | Export-Csv 'C:\Temp\passwords.csv' -NoTypeInformation
Security note: RNGCryptoServiceProvider calls Windows CryptGenRandom — the same entropy source as browser crypto.getRandomValues(). Never use Get-Random for credentials.

Step 2: Import to Active Directory

Import-Module ActiveDirectory
$users = Import-Csv 'C:\Temp\user-list.csv'
$pwds  = Import-Csv 'C:\Temp\passwords.csv'
for ($i = 0; $i -lt $users.Count; $i++) {
    $sec = ConvertTo-SecureString $pwds[$i].Password -AsPlainText -Force
    New-ADUser -SamAccountName $users[$i].SamAccountName `
               -GivenName $users[$i].GivenName -Surname $users[$i].Surname `
               -Name "$($users[$i].GivenName) $($users[$i].Surname)" `
               -Path $users[$i].OU -AccountPassword $sec `
               -ChangePasswordAtLogon $true -Enabled $true
}

Step 3: Secure Distribution

Never email passwords in plain text. Use Microsoft Temporary Access Pass (Entra ID), encrypted PDF with separately communicated passphrase, or one-time-view link services. Set ChangePasswordAtLogon for all accounts. Delete the CSV immediately after distribution.

Step 4: Post-Provisioning Audit

Get-ADUser -Filter {WhenCreated -ge (Get-Date).AddDays(-1)} `
    -Properties PasswordLastSet, WhenCreated |
    Where-Object { $_.PasswordLastSet -lt $_.WhenCreated.AddHours(24) } |
    Select-Object SamAccountName, WhenCreated, PasswordLastSet
