NIST SP 800-63B 2025: What IT Admins Actually Need to Change

28 Apr 2026·8 min·Alex Chen, IT Security Specialist

NIST SP 800-63B published its final 2025 revision in July 2025. Three changes require immediate IT admin action.

Change 1: Minimum Length Now 15 Characters

Computer Configuration > Windows Settings > Security Settings
> Account Policies > Password Policy

Minimum password length:   15    # raised from 8 in most organisations
Maximum password age:       0    # no expiry (see Change 2)
Password complexity:  Disabled   # NIST discourages composition rules
Minimum password age:       1    # prevents immediate re-use
Enforce password history:  24

Change 2: Mandatory Rotation Formally Prohibited

NIST SP 800-63B Section 5.1.1 states verifiers shall not require periodic password changes unless there is evidence of compromise. Set Maximum password age to 0 for all non-PCI accounts. For PCI-DSS accounts: deploy MFA to eliminate the 90-day rotation requirement under PCI-DSS v4.0 Req 8.4.

Change 3: Breach Corpus Checking Now Required

function Test-PasswordBreached {
    param([string]$Password)
    $sha1   = [System.Security.Cryptography.SHA1]::Create()
    $bytes  = [System.Text.Encoding]::UTF8.GetBytes($Password)
    $hash   = [System.BitConverter]::ToString($sha1.ComputeHash($bytes)).Replace('-','')
    $prefix = $hash.Substring(0,5); $suffix = $hash.Substring(5)
    $resp   = Invoke-RestMethod "https://api.pwnedpasswords.com/range/$prefix"
    return ($resp -split "`r`n") -match "^$suffix" | Select-Object -First 1
}

IT Admin Checklist

  • Update minimum password length to 15 in all Group Policy objects
  • Set Maximum password age to 0 for non-PCI accounts
  • Disable complexity rules — replace with length and breach corpus checking
  • Implement HIBP API check at self-service password reset portal
  • Update helpdesk scripts — stop advising users to rotate on a schedule
  • Verify all systems accept passwords up to 64 characters
  • Update user-facing password policy documentation
